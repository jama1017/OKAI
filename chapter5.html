Introduction to Feedforward Neural Networks

Building upon perceptrons, Feedforward Neural Networks (FNN) can complete far more powerful tasks. This chapter presents an anatomy of this simplest type of artificial neural networks. Keep scrolling!

Scroll

To this point, we have only discussed networks with a single perceptron. These perceptron algorithms are not particularly powerful, as they are slow to learn, and cannot learn complex concepts. To improve them, we need to "stack" perceptrons together so that they can form a more complex model.

"Stacking" multiple perceptrons together give us a Feed-forward Neural Network (FNN). Now, instead of only having several inputs, we now have an input layer. The layers that come after this first layer with the exclusion of the last layer, are called hidden layers.

The last layer is called the output layer. We say that a network has n layers when there are n layers that perform computations. That is, there are n-1 hidden layers and 1 output layer.

As an example, let’s say we want to predict the temperature range of tomorrow, meaning that we would like to use an already trained FNN to predict the maximum and minimum temperatures.

We would use the maximum and minimum temperature of today as inputs. The network that we are using is a simple two-layer FNN that takes in two inputs.

These two inputs in the input layer are then sent to 4 perceptrons in the hidden layer. Then, the outputs of the hidden layer are fed into the 2 outputs in the output layer, which generates the maximum and minimum temperature of tomorrow.

In this network, all of the nodes in the hidden layer have a ReLU function, while the last layer is a linear layer (meaning that it has no activation function). But why do we need activation functions when we don’t need to control the range of output here? And what exactly is a ReLU function?

It turns out that, if we have consecutive linear layers, the network will behave exactly the same as a network with a single linear layer (Recall the AND and XOR example in Chapter 2). To make the extra layers count, we need to introduce non-linearity. One of the most popular choices to break linearity is ReLU.

With these activation functions, we can stack more layers together so they can learn to perform more complex tasks, such as recognizing handwritten digits as showcased in Chapter 0.

Now that we have understood the basics of how feedforward neural networks work, we can get our hands a little dirty and look at how we can train a model to classify handwritten digits.

Glossary

Hidden Layer: The middle layers of a FNN that performs computations.

Node: In more complex neural networks, each perceptron is abstracted as a node.

ReLU: Rectified Linear Unit. A type of activation functions most commonly used to break linearity.
