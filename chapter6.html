Feedforward Neural Networks in Action

In this chapter, we will explain how to make a practical network that will recognize handwritten digits, just like the one in Chapter 0. We will be using the MNIST dataset, a dataset of handwritten images from computer scientist Yann LeCun's website.

Each of the images from the MNIST dataset is a 28 pixel by 28 pixel image. These images will be used as training data to teach our FNN how to recognize handwritten digits.

Each pixel has a value from 0 to 1 to indicate how white the pixel is. 0 is black, 1 is white, and anywhere between them is a shade of gray. These 28 × 28 = 784 pixels are flattened into a single array to be fed into the input layer.

The input layer then transforms it into the first hidden layer of size 500, so we have 500 numbers that will represent a transformation of the original image. For a multilayer network to work well, we need to break linearity, and thus we use ReLU as an activation function to achieve that.

These 500 nodes are then feeding into another layer of 10 nodes. Since we are working with classification, this would require us to transform the output of these 10 nodes into probabilities. To do this, we use softmax activation function. In this way, we will get the probability associated with each digit that we can use to calculate loss.

Finally, the output layer will also have 10 nodes, each corresponding to a digit from 0 to 9.

Because of softmax, each final output node now contains a probability value that represents how likely the FNN thinks the input image represents the corresponding digit.

The highest probability, signaling the digit that the model thinks is the most probable, will be our final output classification of the input handwritten digit.

We then split images from MNIST by batches of 100 and feed them into the FNN for training. During training, the model uses existing parameters (weights and biases) to generate a tentative answer. This process is called forward propagation.

After seeing the answer and computing the loss, it goes back into the network and updates the parameters to get even closer to the desired answer. This process is called backpropagation.

To train the model, we would need to first initialize the values inside the model. In this particular case, we will initialize them to random values clustered around 0, which is usually a good starting point.

To train it, we are using Stochastic Gradient Descent (SGD) with a learning rate of 0.7. This learning rate is chosen by experiment, as it seems to work best for our model. Usually, you should try out a few different values before settling on a reasonable one.

After going over the entire dataset for 2 to 3 epochs, our model is good enough to be used. It would be able to achieve around 97% accuracy with only 2 runs over the dataset, and on an average laptop, this would take less than a minute.

This is the FNN that we used in our interactive module in Chapter 0. For those who want to experiment with the model more in depth, we have written an easy-to-understand code that you can play around with online.

Glossary

Softmax: A hyped-up version of sigmoid where it generates the probability of multiple things.

Forward Propagation: The process of making the neural network generate a prediction based on input.

Backpropagation: The process of adjusting parameters to improve the performance of neural networks.
